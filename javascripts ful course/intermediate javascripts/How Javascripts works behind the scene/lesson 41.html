An High-level Overwiew of Javascript

JAVASCRIPT IS A HIGH-LEVEL PROTOTYPE-BASED OBJECT-ORIENTED multi-paradigm interpreted or just-in-time compiled dynamic single-threaded garbage-collector programming language with first-class functions and a non-blocking event loop concurrency model

The breakdown of the definition

1. HIGH-LEVEL
Javascript is one of the high-level language, meaning "it's abstracted from the low-level workings of the computer it is running on". Developer does not have to worry coz everything happens automatically.
2. garbage-collected
Javascript is a refer to as garbage-collector because its monitor memory allocation and determine when a block of allocated memory is no longer needed and reclaim it.
3. interpreted or just-in-time compiled
Javascripts is considered as interpreted language, since the compilation is handled at run time, rather than ahead of time.

4. multi-paradigm
Javascript is considered as multi-paradigm scripting language: which means that it is very dynamic in nature and supports various types of programming styles, such as object-oriented, imperative, and functional programming.
5. PROTOTYPE-BASED OBJECT-ORIENTED
Javascript is a PROTOTYPE-BASED language: meaning object properties and methods can be shared through generalized objects that have the ability to be cloned and extended.

6. first-class functions
Javascript is said be a first-class function because if function in that language are treated like other variables, the functions can be assigned to any other variables or passed as an argument or can be passed as an argument or can be returned by another function.
7. dynamic
Javascript is said to be dynamic because it does not require explicit declaration of the variables before they are used.
8. single-threaded
Javascript is said to be single-threaded language because it's just one line of code may be run at once. in addition, it was only a web browser scripting language created to serve the needs of a single user on a single window of the browser, eliminating the need for multithreading.
9. non-blocking event loop
Javascript is know to be non-blocking event loop because javascript doesn't wait for the response of an API call, an ajax request, an I/O event or a timer but moves on the the other block of code below it.
